# План разработки Backend (Разработчик 1)

## 1. Цели
Реализация надежного и масштабируемого API для управления многомиллионной базой номеров, проектами и интеграциями.

## 2. Стек
- **Язык**: Python 3.12
- **Фреймворк**: FastAPI
- **БД**: PostgreSQL 15+
- **ORM**: SQLAlchemy 2.0 (async)
- **Миграции**: Alembic
- **Очереди задач**: Celery + Redis
- **Auth**: JWT (OAuth2 Password Bearer)

## 3. Схема БД и оптимизация
- **Таблица `contacts`**: 
  - Партиционирование по `created_at` (по месяцам).
  - Индексы: `phone` (unique), `project_id`, `status_id`.
  - Полнотекстовый поиск (GIN индекс на `company_name` и `email`).
- **Таблица `projects`**: хранение лимитов, рабочих дней (JSONB или битовая маска), гео-фильтров.
- **Таблица `audit_logs`**: запись действий (diff JSONB).

## 4. Основные модули
1. **Auth & Roles**:
   - Реализация JWT авторизации.
   - Декораторы для проверки ролей (Admin, Manager).
2. **Projects Module**:
   - CRUD операций.
   - Валидация рабочих часов/дней.
   - Soft delete логика.
3. **Contacts/Numbers Module**:
   - Пагинация (Cursor-based для больших объемов).
   - Фильтрация по сложным критериям (Suppliers, Geo, Days).
   - Дедупликация при вставке.
4. **Integration Module**:
   - SFTP клиент (загрузка/выгрузка файлов).
   - SoftPhone Webhook (обработка POST запросов от телефонии).
5. **Audit & Logging**:
   - Middleware для автоматического логирования изменений сущностей.

## 5. Этапы разработки
1. **Спринт 1**: Инициализация проекта, Docker-compose, Auth, базовые CRUD проектов.
2. **Спринт 2**: Работа с контактами, массовый импорт через Celery, оптимизация БД.
3. **Спринт 3**: SFTP интеграция, SoftPhone Webhook, система аудита.
4. **Спринт 4**: Тестирование, оптимизация запросов, документация API (Swagger).

## 6. ИИ-помощник для разработчика
- Генерация Pydantic схем из моделей БД.
- Написание boilerplate кода для CRUD.
- Генерация тестов (Pytest) для эндпоинтов.
